# 简单支付 MVP Checklist

## 已完成
- 用户、商家、后台三端独立登录和独立界面
- 三端可用独立 URL 入口：`index.html?role=user`、`index.html?role=merchant`、`index.html?role=admin`
- 三端可用独立入口文件：`user.html`、`merchant.html`、`admin.html`
- 三端已准备 PWA manifest：`manifest-user.json`、`manifest-merchant.json`、`manifest-admin.json`
- 用户钱包、充值申请、提现申请、扫码付款、收款码、交易记录
- 商家资料、收款码、订单、退款申请、结算申请、通知、CSV 导出
- 后台用户、商家、资金、退款、结算、交易、财务、风控、KYC、权限、日志、营销、客服、系统配置
- 充值、提现、结算审批凭证和拒绝原因
- 操作日志、财务/交易/订单/日志导出
- Firebase 数据同步、每日限额、冻结限制、备份 JSON
- 后台真实概览和待处理快捷入口

## 还需要打磨
- Firestore 规则最终版收紧
- 后台列表分页，避免免费额度被大量读取消耗
- 数据恢复导入工具
- 用户端/商家端界面拆分成独立入口或独立 App 包
- 更完整的错误提示和空状态中文统一
- 真实支付网关、真实银行出款、正式 KYC 服务
- 正式隐私政策、用户条款、商家协议、合规说明

## 上线前必须确认
- 管理员账号和权限表正确
- Firebase 安全规则不允许普通用户写后台数据
- 所有资金变化只能走审批或交易流程
- 备份文件能定期导出并妥善保存
- 小范围测试账号完成完整流程测试
